После долгих размышлений в голову приходят следующие тезисы 


1) Все объекты конфигурации с прециксом Р должны иметь определённую независимость.
	Их внутреннее изменение и логика работы не должны сказываться на внешних структурах выгрузки
	Запечатывание логики реализации в отдельном механизме смотрится весьма удобно
2) Из п1 следуюет, что необходимо реализовать структуру для выгрузки конфигурации.
   Конфигурация должна биться на определённые файлы с файлом-проектом.

   Conf.xml			-- Корень конфигурации, содержит в себе описание конфигурации и на структуру файлов, которые присоединены к конфугурации
   ---Data			-- Каталог данных 
   ------Component	-- Каталог компонентов
   ---Interface		-- Каталог конфигурации UI
   ---Role
   ...
